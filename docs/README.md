# Project Documentation

This directory contains durable project knowledge that should be available from
a fresh clone.

## Structure

| Directory | Purpose |
|-----------|---------|
| `solutions/` | Troubleshooting write-ups and verified fixes |
| `superpowers/specs/` | Implementation specs and milestone notes |
| `ideation/` | Repo-grounded product and architecture exploration |
| `adr/` | Architecture decision records, when needed |

Local-only artifacts such as downloaded manuals, generated diagnostics, and
temporary reports belong under `local-artifacts/` or `diagnostics/`, both of
which are gitignored.
